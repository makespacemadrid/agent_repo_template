# Handoff entre sesiones

> Tier T2: crear cuando hay múltiples sesiones de trabajo (agentes, developers, o turnos).
> Nota de plantilla: si no hay entradas reales debajo del template, asume que no existe historial de sesiones para este proyecto.

Cada entrada captura el contexto necesario para que la siguiente sesión retome sin pérdida de información.

## Template de entrada

```markdown
### YYYY-MM-DD — [Agente/Persona]

**Objetivo de la sesión**: qué se intentó hacer
**Completado**:
- item 1
- item 2

**Pendiente / en progreso**:
- item con contexto de por qué quedó pendiente

**Archivos tocados**: `ruta/archivo1`, `ruta/archivo2`
**Hallazgos**: descubrimientos relevantes para futuras sesiones
**Decisiones tomadas**: cambios de enfoque o tradeoffs elegidos
**Bloqueos**: qué impidió avanzar (si aplica)
```
